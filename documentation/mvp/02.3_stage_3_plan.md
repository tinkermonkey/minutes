## Goal

Users can identify every speaker by name, merge mistaken duplicates, and listen to voice samples. The local `speakers` table is fully synced with speech-swift on startup and kept current mid-session (already done in Stage 1). This stage adds the UI layer and the four Tauri commands that drive it.

## Type Disambiguation

Three speaker-related types exist across the codebase. They are intentionally separate:

| Type | Defined in | Fields | Used for |
|---|---|---|---|
| `SpeakerNotification` | `src/types/transcript.ts` (Stage 2) | 3 | `new_speaker` event payload — lightweight notification only |
| `Speaker` | `src/types/speaker.ts` (this stage) | 6 | Full speaker record from `get_speakers` Tauri command |
| `SpeakerWithStats` | Rust `db/speakers.rs` (this stage) | 7 | DB query result powering `get_speakers`; adds computed `session_count` |

`SpeakerWithStats` (Rust) serializes to `Speaker` (TypeScript) — `session_count` is included in the TypeScript type. Never substitute `SpeakerNotification` where a full `Speaker` is expected.

---

## Reserved Fields

Two schema fields exist in the system design but have no editing UI in the MVP:

- **`speakers.notes`**: Pulled from speech-swift during startup sync if speech-swift surfaces it; otherwise `NULL`. Read-only in the MVP UI. Reserved for future editing support or external tool integration via the REST API.
- **`sessions.label`**: Always `NULL` in the MVP — no UI for naming sessions. Reserved for future use. Stage 4 displays it where present but will always see `null` in normal MVP use.

### 1. Schema migration (Stage 3 migration)

Add a second rusqlite_migration entry in `db/migrations.rs`. SQLite does not support `ALTER COLUMN`, so recreate both affected tables with `speaker_id` nullable:

```rust
M::up(r#"
    -- Recreate segments with nullable speaker_id
    CREATE TABLE segments_new (
        id              INTEGER PRIMARY KEY,
        session_id      INTEGER NOT NULL REFERENCES sessions(id),
        speaker_id      INTEGER,          -- NULL after speaker deleted
        start_ms        INTEGER NOT NULL,
        end_ms          INTEGER NOT NULL,
        transcript_text TEXT NOT NULL
    );
    INSERT INTO segments_new SELECT * FROM segments;
    DROP TABLE segments;
    ALTER TABLE segments_new RENAME TO segments;

    -- Recreate speaker_samples with nullable speaker_id
    CREATE TABLE speaker_samples_new (
        id          INTEGER PRIMARY KEY,
        speaker_id  INTEGER,              -- NULL after speaker deleted
        session_id  INTEGER NOT NULL REFERENCES sessions(id),
        start_ms    INTEGER NOT NULL,
        end_ms      INTEGER NOT NULL,
        audio_path  TEXT NOT NULL
    );
    INSERT INTO speaker_samples_new SELECT * FROM speaker_samples;
    DROP TABLE speaker_samples;
    ALTER TABLE speaker_samples_new RENAME TO speaker_samples;
"#),
```

This migration runs automatically on next app launch via the existing `migrations().to_latest()` call in `db::open()`.

### 2. speech-swift client extensions — `client::speech_swift`

Add the four speaker management calls to the existing client module:

```rust
#[derive(Debug, serde::Deserialize)]
pub struct SpeakerRecord {
    pub id:           i64,
    pub display_name: Option<String>,
    pub notes:        Option<String>,
}

pub async fn list_speakers(base_url: &str) -> anyhow::Result<Vec<SpeakerRecord>> {
    let resp = reqwest::get(format!("{}/registry/speakers", base_url))
        .await?.error_for_status()?
        .json::<Vec<SpeakerRecord>>().await?;
    Ok(resp)
}

pub async fn rename_speaker(
    base_url: &str,
    speech_swift_id: i64,
    name: &str,
) -> anyhow::Result<()> {
    let client = reqwest::Client::new();
    client.patch(format!("{}/registry/speakers/{}", base_url, speech_swift_id))
        .json(&serde_json::json!({ "display_name": name }))
        .send().await?.error_for_status()?;
    Ok(())
}

pub async fn merge_speakers(
    base_url: &str,
    src_id: i64,
    dst_id: i64,
) -> anyhow::Result<()> {
    let client = reqwest::Client::new();
    client.post(format!("{}/registry/speakers/merge", base_url))
        .json(&serde_json::json!({ "src": src_id, "dst": dst_id }))
        .send().await?.error_for_status()?;
    Ok(())
}

pub async fn delete_speaker(base_url: &str, speech_swift_id: i64) -> anyhow::Result<()> {
    let client = reqwest::Client::new();
    client.delete(format!("{}/registry/speakers/{}", base_url, speech_swift_id))
        .send().await?.error_for_status()?;
    Ok(())
}
```

### 3. DB layer extensions — `db::speakers`

**Extended `Speaker` type** with computed stats:

```rust
#[derive(Debug, serde::Serialize, Clone)]
pub struct SpeakerWithStats {
    pub id:              i64,
    pub speech_swift_id: i64,
    pub display_name:    Option<String>,
    pub notes:           Option<String>,
    pub first_seen_at:   i64,
    pub last_seen_at:    i64,
    pub session_count:   i64,
}
```

**`list_with_stats`** — single query with a subquery for session count:

```rust
pub fn list_with_stats(conn: &Connection) -> anyhow::Result<Vec<SpeakerWithStats>> {
    let mut stmt = conn.prepare(r#"
        SELECT
            s.id, s.speech_swift_id, s.display_name, s.notes,
            s.first_seen_at, s.last_seen_at,
            COUNT(DISTINCT sg.session_id) AS session_count
        FROM speakers s
        LEFT JOIN segments sg ON sg.speaker_id = s.speech_swift_id
        GROUP BY s.id
        ORDER BY s.last_seen_at DESC
    "#)?;
    // map rows → SpeakerWithStats
}
```

**`merge_speaker_local`** — re-point segments + samples, remove source speaker:

```rust
pub fn merge_speaker_local(
    conn: &Connection,
    src_speech_swift_id: i64,
    dst_speech_swift_id: i64,
) -> anyhow::Result<()> {
    conn.execute(
        "UPDATE segments SET speaker_id = ?1 WHERE speaker_id = ?2",
        [dst_speech_swift_id, src_speech_swift_id],
    )?;
    conn.execute(
        "UPDATE speaker_samples SET speaker_id = ?1 WHERE speaker_id = ?2",
        [dst_speech_swift_id, src_speech_swift_id],
    )?;
    conn.execute(
        "DELETE FROM speakers WHERE speech_swift_id = ?1",
        [src_speech_swift_id],
    )?;
    Ok(())
}
```

**`delete_speaker_local`** — disassociate segments + samples, remove speaker:

```rust
pub fn delete_speaker_local(
    conn: &Connection,
    speech_swift_id: i64,
) -> anyhow::Result<()> {
    conn.execute(
        "UPDATE segments SET speaker_id = NULL WHERE speaker_id = ?1",
        [speech_swift_id],
    )?;
    conn.execute(
        "UPDATE speaker_samples SET speaker_id = NULL WHERE speaker_id = ?1",
        [speech_swift_id],
    )?;
    conn.execute(
        "DELETE FROM speakers WHERE speech_swift_id = ?1",
        [speech_swift_id],
    )?;
    Ok(())
}
```

**`get_sample_path`** — return the most recent audio sample path for a speaker (used for playback):

```rust
pub fn get_sample_path(
    conn: &Connection,
    speech_swift_id: i64,
) -> anyhow::Result<Option<String>> {
    conn.query_row(
        "SELECT audio_path FROM speaker_samples
         WHERE speaker_id = ?1
         ORDER BY end_ms DESC LIMIT 1",
        [speech_swift_id],
        |row| row.get(0),
    ).optional().map_err(Into::into)
}
```

### 4. Startup registry sync

Extend the Tauri `.setup()` in `lib.rs` to sync the full speaker registry after the health check:

```rust
tauri::async_runtime::spawn(async move {
    let reachable = client::speech_swift::health_check(&base_url).await;

    // update AppState.speech_swift.reachable (from Stage 0)...

    if reachable {
        // Full registry sync
        if let Ok(speakers) = client::speech_swift::list_speakers(&base_url).await {
            let db = state.db.lock().unwrap();
            let now_ms = unix_ms();
            for record in speakers {
                let (speaker, is_new) = db::speakers::upsert_speaker(
                    &db, record.id, now_ms
                ).unwrap_or_default();
                if record.display_name.is_some() {
                    // Sync display_name from speech-swift into local cache
                    // (catches renames made via other tools)
                    let _ = db.execute(
                        "UPDATE speakers SET display_name = ?1 WHERE speech_swift_id = ?2",
                        rusqlite::params![record.display_name, record.id],
                    );
                }
                if is_new || speaker.display_name.is_none() {
                    events::emit_new_speaker(&handle, speaker.into());
                }
            }
        }
    } else {
        let _ = handle.emit("speech_swift_unreachable", ());
    }
});
```

> The per-chunk upsert from Stage 1 continues to run mid-session. This startup sync only catches changes made externally to speech-swift between sessions.

### 5. Tauri commands — `commands::speakers`

```rust
// commands/speakers.rs

#[tauri::command]
pub fn get_speakers(state: tauri::State<AppState>) -> Result<Vec<SpeakerWithStats>, String> {
    let db = state.db.lock().unwrap();
    db::speakers::list_with_stats(&db).map_err(|e| e.to_string())
}

#[tauri::command]
pub async fn rename_speaker(
    speech_swift_id: i64,
    name: String,
    state: tauri::State<'_, AppState>,
) -> Result<(), String> {
    client::speech_swift::rename_speaker(&state.speech_swift_url, speech_swift_id, &name)
        .await.map_err(|e| e.to_string())?;
    let db = state.db.lock().unwrap();
    db.execute(
        "UPDATE speakers SET display_name = ?1 WHERE speech_swift_id = ?2",
        rusqlite::params![name, speech_swift_id],
    ).map_err(|e| e.to_string())?;
    Ok(())
}

#[tauri::command]
pub async fn merge_speakers(
    src_id: i64,
    dst_id: i64,
    state: tauri::State<'_, AppState>,
) -> Result<(), String> {
    client::speech_swift::merge_speakers(&state.speech_swift_url, src_id, dst_id)
        .await.map_err(|e| e.to_string())?;
    let db = state.db.lock().unwrap();
    db::speakers::merge_speaker_local(&db, src_id, dst_id)
        .map_err(|e| e.to_string())
}

#[tauri::command]
pub async fn delete_speaker(
    speech_swift_id: i64,
    state: tauri::State<'_, AppState>,
) -> Result<(), String> {
    client::speech_swift::delete_speaker(&state.speech_swift_url, speech_swift_id)
        .await.map_err(|e| e.to_string())?;
    let db = state.db.lock().unwrap();
    db::speakers::delete_speaker_local(&db, speech_swift_id)
        .map_err(|e| e.to_string())
}

#[tauri::command]
pub fn get_speaker_sample_path(
    speech_swift_id: i64,
    state: tauri::State<AppState>,
) -> Result<Option<String>, String> {
    let db = state.db.lock().unwrap();
    db::speakers::get_sample_path(&db, speech_swift_id)
        .map_err(|e| e.to_string())
}
```

Register all five commands in `lib.rs`'s `invoke_handler`.

### 8. TanStack Query hooks — `src/hooks/useSpeakers.ts`

```ts
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import { invoke } from '@tauri-apps/api/core';
import type { Speaker } from '../types/speaker';

const SPEAKERS_KEY = ['speakers'] as const;

export function useSpeakers() {
  return useQuery({
    queryKey: SPEAKERS_KEY,
    queryFn: (): Promise<Speaker[]> => invoke('get_speakers'),
  });
}

export function useRenameSpeaker() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: ({ speechSwiftId, name }: { speechSwiftId: number; name: string }) =>
      invoke('rename_speaker', { speechSwiftId, name }),
    onSuccess: () => qc.invalidateQueries({ queryKey: SPEAKERS_KEY }),
  });
}

export function useMergeSpeakers() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: ({ srcId, dstId }: { srcId: number; dstId: number }) =>
      invoke('merge_speakers', { srcId, dstId }),
    onSuccess: () => qc.invalidateQueries({ queryKey: SPEAKERS_KEY }),
  });
}

export function useDeleteSpeaker() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (speechSwiftId: number) =>
      invoke('delete_speaker', { speechSwiftId }),
    onSuccess: () => qc.invalidateQueries({ queryKey: SPEAKERS_KEY }),
  });
}

export function useSpeakerSamplePath(speechSwiftId: number) {
  return useQuery({
    queryKey: ['speaker_sample', speechSwiftId],
    queryFn: (): Promise<string | null> =>
      invoke('get_speaker_sample_path', { speechSwiftId }),
    enabled: speechSwiftId > 0,
  });
}
```

All mutating hooks invalidate `['speakers']` on success — the list re-fetches automatically.

### 11. `SpeakerNameField` — inline rename

**`src/components/speakers/SpeakerNameField.tsx`**

States: `display` (showing name or "Unknown") and `editing` (text input visible).

```
display state:
  [Speaker Name ✏]   ← click anywhere or pencil icon → editing state

editing state:
  [__input______] [✓] [✕]
```

- Click anywhere on the name text (or the pencil icon) → switch to editing state, pre-fill input with current `display_name ?? ''`.
- **Confirm** (✓ or Enter): call `renameSpeaker.mutateAsync({ speechSwiftId, name: inputValue.trim() })`; return to display state on success.
- **Cancel** (✕ or Escape): discard input, return to display state.
- While mutation is pending: disable input and buttons, show spinner on ✓.
- Empty name: disable ✓ button (name cannot be blank).
- Display state shows "Unknown" in muted italic when `display_name` is null.

### 12. `AudioPlayer` component

**`src/components/speakers/AudioPlayer.tsx`**

```tsx
interface Props {
  speechSwiftId: number;
}
```

On mount, fetch the sample path via `useSpeakerSamplePath`. If `null`, render nothing (no samples yet for this speaker).

When a path is available, render a play button (Flowbite `Button`, `size="xs"`, icon-only with headphone icon):
- On click: convert the path to an asset URL with `convertFileSrc(path)` from `@tauri-apps/api/core`.
- Construct an `HTMLAudioElement` in JS (`new Audio(url)`) and call `.play()`.
- While playing: show a stop button (swap icons). Click stop → `.pause()` + reset.
- On playback end: revert to play button automatically.

Do not use an `<audio>` DOM element — manage the `Audio` instance in a `useRef` to avoid layout shifts and keep the component minimal.

```ts
const audioRef = useRef<HTMLAudioElement | null>(null);

function handlePlay() {
  const url = convertFileSrc(samplePath);
  audioRef.current = new Audio(url);
  audioRef.current.play();
  audioRef.current.onended = () => setPlaying(false);
  setPlaying(true);
}

function handleStop() {
  audioRef.current?.pause();
  setPlaying(false);
}
```

### 13. Merge flow

**Merge state** (local to `SpeakersRoute`):

```ts
type MergeState =
  | { phase: 'idle' }
  | { phase: 'selecting'; srcId: number }
  | { phase: 'confirming'; srcId: number; dstId: number };
```

**`MergeSelectButton`** adapts per card based on merge state:

| Merge phase | This card is `src` | This card is `dst` candidate | Other cards |
|---|---|---|---|
| `idle` | — | — | "Select" button |
| `selecting` (src chosen) | "Cancel" button | "Merge with [src name]" button | "Select" button (changes src) |
| `confirming` | — | — | all disabled |

**`MergeConfirmModal`** (Flowbite `Modal`):

Shows when `phase === 'confirming'`:

```
Merge Speakers

  [src name] will be merged into [dst name].
  All transcript segments attributed to [src name] will be
  re-attributed to [dst name]. This cannot be undone.

  [Cancel]   [Merge]
```

On confirm: call `mergeSpeakers.mutateAsync({ srcId, dstId })` → close modal → reset merge state to `idle`.

On cancel: reset to `{ phase: 'selecting', srcId }` (stay in select mode, user can pick a different target).

### 15. Full `SpeakersRoute` assembly

**`src/routes/speakers.tsx`**

```tsx
export function SpeakersRoute() {
  const { data: speakers = [], isLoading } = useSpeakers();
  const [mergeState, setMergeState] = useState<MergeState>({ phase: 'idle' });
  const [deleteTarget, setDeleteTarget] = useState<Speaker | null>(null);
  const mergeSpeakers = useMergeSpeakers();
  const deleteSpeaker = useDeleteSpeaker();

  if (isLoading) return <Spinner />;

  return (
    <div className="p-6 flex flex-col gap-4">
      <SpeakerListHeader count={speakers.length} />

      {speakers.length === 0 && (
        <p className="text-gray-500">No speakers yet. Start a recording session to detect speakers.</p>
      )}

      <div className="flex flex-col gap-3">
        {speakers.map(speaker => (
          <SpeakerCard
            key={speaker.id}
            speaker={speaker}
            mergeState={mergeState}
            onMergeSelect={(id) => handleMergeSelect(speaker, id)}
            onDeleteClick={() => setDeleteTarget(speaker)}
          />
        ))}
      </div>

      {mergeState.phase === 'confirming' && (
        <MergeConfirmModal
          src={speakers.find(s => s.speech_swift_id === mergeState.srcId)!}
          dst={speakers.find(s => s.speech_swift_id === mergeState.dstId)!}
          onConfirm={async () => {
            await mergeSpeakers.mutateAsync({ srcId: mergeState.srcId, dstId: mergeState.dstId });
            setMergeState({ phase: 'idle' });
          }}
          onCancel={() => setMergeState({ phase: 'selecting', srcId: mergeState.srcId })}
        />
      )}

      {deleteTarget && (
        <DeleteConfirmModal
          speaker={deleteTarget}
          onConfirm={async () => {
            await deleteSpeaker.mutateAsync(deleteTarget.speech_swift_id);
            setDeleteTarget(null);
          }}
          onCancel={() => setDeleteTarget(null)}
        />
      )}
    </div>
  );
}
```
