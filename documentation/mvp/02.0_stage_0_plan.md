# Stage 0 — Foundation Plan

## UX Reference

**Design file**: `documentation/minutes-app-screens.pen`
**Relevant frames**: All four screens — the **sidebar nav** (left panel) is implemented in this stage as the root layout and must match the design across every screen.

When building the root layout and sidebar, use the `ux-design-architect` agent to open the design file and inspect the sidebar frame before writing any component code:
- Sidebar width: 220px, thin right border
- Nav links: Record · Speakers · Sessions · Search (active state: `bg-blue-50 text-blue-700 font-semibold`)
- Settings gear icon at the bottom of the sidebar
- Consult the design for exact spacing, typography scale, and active/inactive link styles before building `__root.tsx`

## Goal

The app boots with a working UI shell, a fully migrated SQLite schema (including the sqlite-vec extension), and a confirmed speech-swift connectivity check. Every subsequent stage depends on this foundation being solid.

---

1. Frontend UI shell with four stub routes wired up in TanStack Router
2. Flowbite React + Tailwind CSS, TanStack Table, and TanStack Virtual installed and rendering
3. SQLite initialized at app startup via rusqlite_migration with all five tables from the schema
4. `sqlite-vec` extension loaded at DB init, `segment_embeddings` virtual table created
5. Tauri capabilities declared: microphone access, file dialog
6. `GET /health` called on speech-swift at launch; result stored in Tauri app state and exposed to the frontend

### 3. Rust — SQLite initialization with rusqlite_migration

**Cargo.toml additions**

```toml
[dependencies]
rusqlite = { version = "...", features = ["bundled"] }
rusqlite_migration = "..."
sqlite-vec = "..."          # provides the extension load helper
```

**Module: `src-tauri/src/db/migrations.rs`**

Define all migrations as a `Migrations` constant. Migration 1 creates all five tables:

```rust
use rusqlite_migration::{Migrations, M};

pub fn migrations() -> Migrations<'static> {
    Migrations::new(vec![
        M::up(r#"
            CREATE TABLE sessions (
                id          INTEGER PRIMARY KEY,
                created_at  INTEGER NOT NULL,
                label       TEXT,
                duration_ms INTEGER,
                source      TEXT NOT NULL
            );

            CREATE TABLE segments (
                id              INTEGER PRIMARY KEY,
                session_id      INTEGER NOT NULL REFERENCES sessions(id),
                speaker_id      INTEGER NOT NULL,
                start_ms        INTEGER NOT NULL,
                end_ms          INTEGER NOT NULL,
                transcript_text TEXT NOT NULL
            );

            CREATE TABLE speaker_samples (
                id          INTEGER PRIMARY KEY,
                speaker_id  INTEGER NOT NULL,
                session_id  INTEGER NOT NULL REFERENCES sessions(id),
                start_ms    INTEGER NOT NULL,
                end_ms      INTEGER NOT NULL,
                audio_path  TEXT NOT NULL
            );

            CREATE TABLE speakers (
                id              INTEGER PRIMARY KEY,
                speech_swift_id INTEGER NOT NULL UNIQUE,
                display_name    TEXT,
                notes           TEXT,
                first_seen_at   INTEGER NOT NULL,
                last_seen_at    INTEGER NOT NULL
            );
        "#),
    ])
}
```

**Module: `src-tauri/src/db/mod.rs`**

Open the SQLite connection, load the sqlite-vec extension, run migrations, then create the `segment_embeddings` virtual table if it does not already exist:

```rust
use rusqlite::Connection;
use sqlite_vec::sqlite3_vec_init;

pub fn open(db_path: &Path) -> anyhow::Result<Connection> {
    let mut conn = Connection::open(db_path)?;

    // Load sqlite-vec
    unsafe {
        conn.load_extension_enable()?;
        conn.load_extension_from_init_fn(sqlite3_vec_init, None)?;
        conn.load_extension_disable()?;
    }

    // Apply schema migrations
    migrations::migrations().to_latest(&mut conn)?;

    // Create vector table (idempotent — not managed by rusqlite_migration
    // because virtual tables cannot be rolled back cleanly)
    conn.execute_batch(r#"
        CREATE VIRTUAL TABLE IF NOT EXISTS segment_embeddings USING vec0(
            segment_id INTEGER PRIMARY KEY,
            embedding  FLOAT[384]
        );
    "#)?;

    Ok(conn)
}
```

> **Why separate the virtual table**: `vec0` virtual tables do not support standard `DROP TABLE` rollback semantics, so they are created outside the migration runner using `IF NOT EXISTS`.

**Wire into Tauri setup (`lib.rs`)**

Resolve the DB path using Tauri's `app_data_dir()`, call `db::open()`, and store the `Connection` in `tauri::Manager` app state (wrapped in a `Mutex`):

```rust
tauri::Builder::default()
    .setup(|app| {
        let data_dir = app.path().app_data_dir()?;
        std::fs::create_dir_all(&data_dir)?;
        let conn = db::open(&data_dir.join("minutes.db"))?;
        app.manage(DbState(Mutex::new(conn)));
        Ok(())
    })
```

**Verify**: Launch the app; confirm `minutes.db` is created in the correct data directory. Open the database with a SQLite browser and verify all five tables (including the `segment_embeddings` virtual table) exist.


**`tauri.conf.json`** — ensure the `dialog` plugin is listed under `plugins`:

```json

    "dialog": {}
  }
}
```

**Microphone entitlement (macOS)**

In `src-tauri/Info.plist` (create if absent):

```xml
<key>NSMicrophoneUsageDescription</key>
<string>Minutes needs microphone access to record and transcribe conversations.</string>
```

Also add the entitlement to `src-tauri/entitlements.plist`:

```xml
<key>com.apple.security.device.audio-input</key>
<true/>
```

And reference it in `tauri.conf.json` under `bundle.macOS.entitlements`.

**Verify**: Build and run on macOS; confirm the app requests microphone permission on first launch.

### 5. speech-swift health check

**App state type (`src-tauri/src/state.rs`)**

```rust
pub struct SpeechSwiftStatus {
    pub reachable: bool,
}

pub struct AppState {
    pub db: Mutex<Connection>,
    pub speech_swift: Mutex<SpeechSwiftStatus>,
    pub speech_swift_url: String,   // default "http://localhost:8080"
}
```

**HTTP client (`src-tauri/src/client/speech_swift.rs`)**

```rust
pub async fn health_check(base_url: &str) -> bool {
    let url = format!("{}/health", base_url);
    reqwest::get(&url).await.map(|r| r.status().is_success()).unwrap_or(false)
}
```

**Tauri setup (`lib.rs`)**

Call the health check asynchronously in `.setup()`. Store the result immediately in `AppState`. Emit a `speech_swift_unreachable` event when the check fails so the frontend can surface a warning banner from the first render:

```rust
.setup(|app| {
    // DB init (see §3) ...

    let handle = app.handle().clone();
    let base_url = "http://localhost:8080".to_string();

    tauri::async_runtime::spawn(async move {
        let reachable = client::speech_swift::health_check(&base_url).await;
        {
            let state = handle.state::<AppState>();
            state.speech_swift.lock().unwrap().reachable = reachable;
        }
        if !reachable {
            let _ = handle.emit("speech_swift_unreachable", ());
        }
    });

    Ok(())
})
```

**Tauri command**

Expose the cached status so the frontend can read it on initial load (before the async check completes, the frontend listens for the event):

```rust
#[tauri::command]
fn get_speech_swift_status(state: tauri::State<AppState>) -> bool {
    state.speech_swift.lock().unwrap().reachable
}
```

**Frontend integration**

In the root layout component (`__root.tsx`):

1. Call `invoke<boolean>('get_speech_swift_status')` on mount via TanStack Query.
2. Subscribe to the `speech_swift_unreachable` Tauri event via `listen()`.
3. If unreachable, show a persistent Flowbite `Alert` (warning style) in the layout header: "speech-swift is unreachable — recording is disabled."

No action buttons or retry logic in Stage 0; that is deferred to Stage 6.

**Verify**: Start the app without speech-swift running; confirm the warning banner appears. Start speech-swift first; confirm no banner.
