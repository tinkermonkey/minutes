### 3. Audio device commands â€” `commands::devices`

**`src-tauri/src/commands/devices.rs`** (new file)

```rust
use cpal::traits::{DeviceTrait, HostTrait};

#[derive(Debug, serde::Serialize)]
pub struct AudioDevice {
    pub name:       String,
    pub is_default: bool,
}

#[tauri::command]
pub fn get_audio_devices(
    state: tauri::State<AppState>,
) -> Result<Vec<AudioDevice>, String> {
    let host = cpal::default_host();

    let default_name = host.default_input_device()
        .and_then(|d| d.name().ok());

    let devices = host.input_devices()
        .map_err(|e| e.to_string())?
        .filter_map(|d| d.name().ok().map(|name| {
            let is_default = default_name.as_deref() == Some(&name);
            AudioDevice { name, is_default }
        }))
        .collect();

    Ok(devices)
}

#[tauri::command]
pub fn set_audio_device(
    device_name: String,
    state: tauri::State<AppState>,
) -> Result<(), String> {
    // Persist to DB
    let db = state.db.lock().unwrap();
    db::settings::set(&db, "preferred_audio_device", &device_name)
        .map_err(|e| e.to_string())?;
    drop(db);

    // Update in-memory preference
    *state.preferred_device.lock().unwrap() = Some(device_name);
    Ok(())
}
```

### 4. On-demand health check â€” `commands::health`

```rust
#[tauri::command]
pub async fn retry_health_check(
    state:  tauri::State<'_, AppState>,
    app:    tauri::AppHandle,
) -> Result<bool, String> {
    let reachable = client::speech_swift::health_check(&state.speech_swift_url).await;
    *state.speech_swift.lock().unwrap() = SpeechSwiftStatus { reachable };
    if !reachable {
        let _ = app.emit("speech_swift_unreachable", ());
    } else {
        let _ = app.emit("speech_swift_reachable", ());   // new event
    }
    Ok(reachable)
}
```

});

### 12. Axum REST API â€” `api` module

The system design specifies a read-only REST API on `127.0.0.1:8765` for external tooling. It shares the same SQLite data as the Tauri backend but runs as an independent axum server on a separate tokio task. No new functionality is required â€” only a thin HTTP surface over the existing DB query functions.

**Why Stage 6**: The REST API has no user-visible effect during Stages 1â€“5. Stage 6 is the correct place since all DB query functions it needs are complete by then.

**DB access**: Open a second SQLite connection in **read-only mode** for the axum server. This avoids write-lock contention with the recording pipeline while keeping the axum server fully independent:

```rust
// db/mod.rs â€” add alongside the existing open() function
pub fn open_readonly(db_path: &Path) -> anyhow::Result<Connection> {
    let conn = Connection::open_with_flags(
        db_path,
        rusqlite::OpenFlags::SQLITE_OPEN_READ_ONLY | rusqlite::OpenFlags::SQLITE_OPEN_NO_MUTEX,
    )?;
    unsafe {
        conn.load_extension_enable()?;
        conn.load_extension_from_init_fn(sqlite3_vec_init, None)?;
        conn.load_extension_disable()?;
    }
    Ok(conn)
}
```

**Module layout**:

```
src-tauri/src/api/
  mod.rs        # axum router definition + server spawn
  sessions.rs   # GET /sessions, GET /sessions/:id/segments
  speakers.rs   # GET /speakers
  search.rs     # GET /search?q=&speaker_id=&start=&end=
```

**Axum state**:

```rust
// api/mod.rs
#[derive(Clone)]
pub struct ApiState {
    pub db_path: PathBuf,
}
```

Each request handler opens a fresh read-only connection from `ApiState.db_path`. Connection overhead is negligible for the expected request rate (external scripts, not high-frequency clients).

**Router**:

```rust
pub fn router(state: ApiState) -> axum::Router {
    axum::Router::new()
        .route("/sessions",                    get(sessions::list_sessions))
        .route("/sessions/:id/segments",       get(sessions::get_segments))
        .route("/speakers",                    get(speakers::list_speakers))
        .route("/search",                      get(search::search))
        .with_state(state)
}
```

**Server spawn** in `lib.rs` setup, after DB init:

```rust
let api_state = api::ApiState { db_path: data_dir.join("minutes.db") };
tauri::async_runtime::spawn(async move {
    let app = api::router(api_state);
    let listener = tokio::net::TcpListener::bind("127.0.0.1:8765").await
        .expect("failed to bind REST API port");
    axum::serve(listener, app).await
        .expect("REST API server failed");
});
```

**Endpoint implementations** (reuse existing DB functions):

```rust
// api/sessions.rs
pub async fn list_sessions(
    State(state): State<ApiState>,
    Query(params): Query<HashMap<String, String>>,
) -> impl IntoResponse {
    let conn = db::open_readonly(&state.db_path).unwrap();
    let filter = SessionFilter::from_query_params(&params); // parse page, sort, dates
    match db::sessions::list_sessions(&conn, &filter) {
        Ok(page) => Json(page).into_response(),
        Err(e)   => (StatusCode::INTERNAL_SERVER_ERROR, e.to_string()).into_response(),
    }
}

pub async fn get_segments(
    State(state): State<ApiState>,
    Path(id): Path<i64>,
) -> impl IntoResponse {
    let conn = db::open_readonly(&state.db_path).unwrap();
    match db::segments::get_segments_with_speakers(&conn, id) {
        Ok(segs) => Json(segs).into_response(),
        Err(e)   => (StatusCode::INTERNAL_SERVER_ERROR, e.to_string()).into_response(),
    }
}

// api/speakers.rs
pub async fn list_speakers(State(state): State<ApiState>) -> impl IntoResponse {
    let conn = db::open_readonly(&state.db_path).unwrap();
    match db::speakers::list_with_stats(&conn) {
        Ok(s) => Json(s).into_response(),
        Err(e) => (StatusCode::INTERNAL_SERVER_ERROR, e.to_string()).into_response(),
    }
}

// api/search.rs
pub async fn search(
    State(state): State<ApiState>,
    Query(params): Query<HashMap<String, String>>,
) -> impl IntoResponse {
    let q = match params.get("q") {
        Some(q) if !q.is_empty() => q.clone(),
        _ => return (StatusCode::BAD_REQUEST, "missing q parameter").into_response(),
    };
    let filters = SearchFilters {
        speaker_id: params.get("speaker_id").and_then(|v| v.parse().ok()),
        start_date: params.get("start").and_then(|v| v.parse().ok()),
        end_date:   params.get("end").and_then(|v| v.parse().ok()),
        limit:      None,
    };
    // embed query synchronously â€” acceptable latency for external tooling
    let embedding = match embed::embed(&q) {
        Ok(e)  => e,
        Err(e) => return (StatusCode::SERVICE_UNAVAILABLE, e.to_string()).into_response(),
    };
    let conn = db::open_readonly(&state.db_path).unwrap();
    match db::search::search_segments(&conn, &embedding, &filters) {
        Ok(results) => Json(results).into_response(),
        Err(e)      => (StatusCode::INTERNAL_SERVER_ERROR, e.to_string()).into_response(),
    }
}
```

**No auth, no CORS headers** â€” the server binds only to `127.0.0.1`, so only local processes can reach it.

### 13. speech-swift port configurability

The system design notes that speech-swift's port is configurable (default 8080). Stage 0 hard-codes `http://localhost:8080`. Stage 6 surfaces port configuration in the Settings drawer so users running speech-swift on a non-default port do not need to rebuild the app.

**DB key**: `speech_swift_port` in the `settings` table (already added in this stage's migration).

};
app_state.speech_swift_url = format!("http://localhost:{}", port);

interface Props {
  message:  string;
  onRetry?: () => void;
}

export function QueryError({ message, onRetry }: Props) {
  return (
    <Alert color="failure" className="m-4">
      <div className="flex flex-col gap-2">
        <span className="font-medium">Failed to load data</span>
        <span className="text-sm font-mono">{message}</span>
        {onRetry && (
          <Button size="xs" color="failure" onClick={onRetry}>Retry</Button>
        )}
      </div>
    </Alert>
  );
}
