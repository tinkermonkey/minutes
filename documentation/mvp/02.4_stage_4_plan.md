## Goal

Users can revisit any past session and re-read its full transcript. The `/sessions` route shows a paginated, sortable, date-filterable session list with participant chips. Clicking a session opens a read-only transcript view with speaker labels resolved from the local cache.

### 2. DB layer — `db::sessions`

**`list_sessions`**

Two queries: a count query and a data query. Build the `WHERE` clause dynamically based on filter fields.

```rust
pub fn list_sessions(
    conn: &Connection,
    filter: &SessionFilter,
) -> anyhow::Result<SessionsPage> {
    let where_clause = build_where(filter);   // e.g. "WHERE created_at >= ? AND created_at <= ?"
    let params        = build_params(filter);  // bound values in order

    let order = match (&filter.sort_by, &filter.sort_dir) {
        (SortBy::Date,     SortDir::Desc) => "created_at DESC",
        (SortBy::Date,     SortDir::Asc)  => "created_at ASC",
        (SortBy::Duration, SortDir::Desc) => "duration_ms DESC NULLS LAST",
        (SortBy::Duration, SortDir::Asc)  => "duration_ms ASC NULLS LAST",
    };
    let offset = ((filter.page - 1) * filter.page_size) as i64;

    let total_count: i64 = conn.query_row(
        &format!("SELECT COUNT(*) FROM sessions {where_clause}"),
        params.as_slice(),
        |r| r.get(0),
    )?;

    let mut stmt = conn.prepare(&format!(
        "SELECT id, created_at, label, duration_ms, source
         FROM sessions {where_clause}
         ORDER BY {order}
         LIMIT ?{next} OFFSET ?{next2}",
        // bind page_size and offset after the where params
    ))?;

    let mut sessions: Vec<SessionRow> = stmt.query_map(...)?.collect::<Result<_, _>>()?;

    // Hydrate participants for each session on this page
    for session in &mut sessions {
        session.participants = get_participants(conn, session.id)?;
    }

    Ok(SessionsPage { sessions, total_count })
}
```

**`get_participants`** (private helper)

```rust
fn get_participants(conn: &Connection, session_id: i64) -> anyhow::Result<Vec<SessionParticipant>> {
    let mut stmt = conn.prepare(
        "SELECT DISTINCT sg.speaker_id, sp.display_name
         FROM segments sg
         LEFT JOIN speakers sp ON sp.speech_swift_id = sg.speaker_id
         WHERE sg.session_id = ?1 AND sg.speaker_id IS NOT NULL"
    )?;
    stmt.query_map([session_id], |row| Ok(SessionParticipant {
        speech_swift_id: row.get(0)?,
        display_name:    row.get(1)?,
    }))?.collect::<Result<_, _>>().map_err(Into::into)
}
```

> The N+1 here (one participants query per session on the page) is intentional and acceptable: page size is small (≤ 20) and sessions are not queried at high frequency.

**`get_session_by_id`**

```rust
pub fn get_session_by_id(
    conn: &Connection,
    session_id: i64,
) -> anyhow::Result<Option<SessionRow>> {
    let result = conn.query_row(
        "SELECT id, created_at, label, duration_ms, source FROM sessions WHERE id = ?1",
        [session_id],
        |row| Ok(/* map to SessionRow without participants */),
    ).optional()?;

    if let Some(mut session) = result {
        session.participants = get_participants(conn, session.id)?;
        Ok(Some(session))
    } else {
        Ok(None)
    }
}
```

### 3. DB layer — `db::segments` extension

Add `get_segments_with_speakers` to the existing `db/segments.rs`:

```rust
#[derive(Debug, serde::Serialize)]
pub struct SegmentWithSpeaker {
    pub id:              i64,
    pub session_id:      i64,
    pub speaker_id:      Option<i64>,
    pub start_ms:        i64,
    pub end_ms:          i64,
    pub transcript_text: String,
    pub display_name:    Option<String>,  // from speakers table
    pub speaker_label:   Option<String>,  // raw speech-swift label (not stored — derive from speaker_id if null)
}

pub fn get_segments_with_speakers(
    conn: &Connection,
    session_id: i64,
) -> anyhow::Result<Vec<SegmentWithSpeaker>> {
    let mut stmt = conn.prepare(
        "SELECT sg.id, sg.session_id, sg.speaker_id, sg.start_ms, sg.end_ms,
                sg.transcript_text, sp.display_name
         FROM segments sg
         LEFT JOIN speakers sp ON sp.speech_swift_id = sg.speaker_id
         WHERE sg.session_id = ?1
         ORDER BY sg.start_ms ASC"
    )?;
    stmt.query_map([session_id], |row| Ok(SegmentWithSpeaker {
        id:              row.get(0)?,
        session_id:      row.get(1)?,
        speaker_id:      row.get(2)?,
        start_ms:        row.get(3)?,
        end_ms:          row.get(4)?,
        transcript_text: row.get(5)?,
        display_name:    row.get(6)?,
        speaker_label:   None,   // frontend derives fallback label from speaker_id
    }))?.collect::<Result<_, _>>().map_err(Into::into)
}
```

> `speaker_label` is not stored in SQLite — speech-swift owns it. The frontend falls back to `"Speaker {speaker_id}"` when `display_name` is null. This is consistent with Stage 2's behaviour.

### 6. Formatting helpers — `src/lib/format.ts`

```ts
/** Format a unix-ms timestamp as "Apr 11, 2026" */
export function formatDate(ms: number): string {
  return new Intl.DateTimeFormat('en-US', {
    month: 'short', day: 'numeric', year: 'numeric',
  }).format(new Date(ms));
}

/** Format a unix-ms timestamp as "2:34 PM" */
export function formatTime(ms: number): string {
  return new Intl.DateTimeFormat('en-US', {
    hour: 'numeric', minute: '2-digit',
  }).format(new Date(ms));
}

/** Format duration_ms as "1h 23m" or "45m" or "< 1m" */
export function formatDuration(ms: number | null): string {
  if (ms === null) return '—';
  const totalSeconds = Math.floor(ms / 1000);
  const h = Math.floor(totalSeconds / 3600);
  const m = Math.floor((totalSeconds % 3600) / 60);
  if (h > 0) return `${h}h ${m}m`;
  if (m > 0) return `${m}m`;
  return '< 1m';
}

/** Format start_ms (relative to session) as "m:ss" — reused from Stage 2 */
export function formatRelativeTime(ms: number): string {
  const totalSeconds = Math.floor(ms / 1000);
  const m = Math.floor(totalSeconds / 60);
  const s = totalSeconds % 60;
  return `${m}:${String(s).padStart(2, '0')}`;
}
```

### 7. TanStack Query hooks — `src/hooks/useSessions.ts`

```ts
import { useQuery } from '@tanstack/react-query';
import { invoke } from '@tauri-apps/api/core';
import type { SessionFilter, SessionsPage, Session, SegmentWithSpeaker } from '../types/session';

export function useSessions(filter: SessionFilter) {
  return useQuery({
    queryKey: ['sessions', filter],
    queryFn: (): Promise<SessionsPage> => invoke('get_sessions', { filter }),
    placeholderData: (prev) => prev,   // keep previous page visible while fetching next
  });
}

export function useSession(sessionId: number) {
  return useQuery({
    queryKey: ['session', sessionId],
    queryFn: (): Promise<Session | null> => invoke('get_session', { sessionId }),
    enabled: sessionId > 0,
  });
}

export function useSegments(sessionId: number) {
  return useQuery({
    queryKey: ['segments', sessionId],
    queryFn: (): Promise<SegmentWithSpeaker[]> => invoke('get_segments', { sessionId }),
    enabled: sessionId > 0,
  });
}
```

`placeholderData: (prev) => prev` (TanStack Query v5 API) keeps the previous page's data visible during page transitions, preventing a blank flash.

#### TanStack Table setup

```ts
import {
  useReactTable,
  getCoreRowModel,
  flexRender,
  createColumnHelper,
} from '@tanstack/react-table';
import type { Session } from '../types/session';

const col = createColumnHelper<Session>();

const columns = [
  col.accessor('created_at', {
    header: () => <SortableHeader label="Date" field="date" filter={filter} onSort={handleSort} />,
    cell: info => (
      <div>
        <div className="font-medium">{formatDate(info.getValue())}</div>
        <div className="text-sm text-gray-500">{formatTime(info.getValue())}</div>
      </div>
    ),
  }),
  col.accessor('duration_ms', {
    header: () => <SortableHeader label="Duration" field="duration" filter={filter} onSort={handleSort} />,
    cell: info => formatDuration(info.getValue()),
  }),
  col.accessor('participants', {
    header: 'Participants',
    enableSorting: false,
    cell: info => <ParticipantChips participants={info.getValue()} />,
  }),
  col.accessor('source', {
    header: 'Source',
    enableSorting: false,
    cell: info => <SourceBadge source={info.getValue()} />,
  }),
];

const table = useReactTable({
  data:              data?.sessions ?? [],
  columns,
  getCoreRowModel:   getCoreRowModel(),
  manualPagination:  true,
  pageCount:         Math.ceil((data?.total_count ?? 0) / filter.page_size),
  state: {
    pagination: { pageIndex: filter.page - 1, pageSize: filter.page_size },
  },
  onPaginationChange: updater => {
    const next = typeof updater === 'function'
      ? updater({ pageIndex: filter.page - 1, pageSize: filter.page_size })
      : updater;
    setFilter(prev => ({ ...prev, page: next.pageIndex + 1 }));
  },
});
```

`manualPagination: true` tells TanStack Table that the data array is already the current page — it should not slice it. `pageCount` is computed from the server-returned `total_count`.

#### Sort handler

```ts
function handleSort(field: SortBy) {
  setFilter(prev => ({
    ...prev,
    sort_by:  field,
    sort_dir: prev.sort_by === field && prev.sort_dir === 'desc' ? 'asc' : 'desc',
    page:     1,
  }));
}
```

```tsx
export function SessionListRoute() {
  const [filter, setFilter] = useState<SessionFilter>({ /* defaults */ });
  const { data, isLoading, isFetching } = useSessions(filter);
  const navigate = useNavigate();

  // TanStack Table setup (see §8)

            <table className="w-full text-sm">
              <thead>
                {table.getHeaderGroups().map(hg => (
                  <tr key={hg.id}>
                    {hg.headers.map(h => (
                      <th key={h.id} className="text-left p-3 border-b">
                        {flexRender(h.column.columnDef.header, h.getContext())}
                      </th>
                    ))}
                  </tr>
                ))}
              </thead>
              <tbody>
                {table.getRowModel().rows.map(row => (
                  <tr
                    key={row.id}
                    className="cursor-pointer hover:bg-gray-50 border-b"
                    onClick={() => navigate({
                      to: '/sessions/$sessionId',
                      params: { sessionId: String(row.original.id) },
                    })}
                  >
                    {row.getVisibleCells().map(cell => (
                      <td key={cell.id} className="p-3">
                        {flexRender(cell.column.columnDef.cell, cell.getContext())}
                      </td>
                    ))}
                  </tr>
                ))}
              </tbody>
            </table>
          </div>

          {data && data.total_count > filter.page_size && (
            <SessionPagination table={table} totalCount={data.total_count} />
          )}

          {data?.sessions.length === 0 && (
            <p className="text-gray-500 text-center py-12">
              No sessions found. {filter.start_date || filter.end_date ? 'Try adjusting the date filter.' : 'Start a recording to create your first session.'}
            </p>
          )}
        </>
      )}
    </div>
  );
}
```

The `opacity-60 pointer-events-none` overlay (rather than a full spinner) during `isFetching` lets the user see the previous page while the next page loads — a smoother experience enabled by `placeholderData`.

### 17. `TranscriptReplayPanel` component

**`src/components/sessions/TranscriptReplayPanel.tsx`**

```tsx
interface Props {
  segments:     SegmentWithSpeaker[];
  sessionStart: number;   // unix ms — used to compute relative timestamps
}
```

Renders the full segment list in a vertically scrolling container. Reuses `SegmentCard` from Stage 2 with a small adaptation: the `start_ms` passed to `TimestampLabel` is already session-relative (stored that way in Stage 1), so no arithmetic is needed.

**Speaker label fallback**: when `display_name` is null and `speaker_id` is non-null, show `"Speaker ${speaker_id}"`. When `speaker_id` is null (speaker was deleted), show `"Unknown"` in muted italic. Pass a derived `label` prop to `SegmentCard`:

```ts
function resolveLabel(seg: SegmentWithSpeaker): string {
  if (seg.display_name) return seg.display_name;
  if (seg.speaker_id !== null) return `Speaker ${seg.speaker_id}`;
  return 'Unknown';
}
```

**No auto-scroll** in replay mode — the user is reading, not watching a live feed.

**Empty state**: if `segments` is empty, show "No transcript available for this session."

**Performance**: TanStack Virtual is installed. For sessions with more than 200 segments, switch from a plain `map` to `useVirtualizer`:

```ts
import { useVirtualizer } from '@tanstack/react-virtual';

const virtualizer = useVirtualizer({
  count:          segments.length,
  getScrollElement: () => containerRef.current,
  estimateSize:   () => 80,   // estimated card height in px
});
```

Use virtual rendering unconditionally for simplicity — it works correctly for both short and long lists.
