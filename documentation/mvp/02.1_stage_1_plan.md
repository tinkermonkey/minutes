# Stage 1 — Audio Pipeline + Persistence Plan

## Goal

The full Rust pipeline from microphone to SQLite is operational. A user can call `start_session` and `stop_session` via Tauri commands; voiced audio is captured, chunked, transcribed by speech-swift, persisted in SQLite, embedded by fastembed-rs, and emitted as Tauri events — all without any UI.

## Tasks

### 1. Audio capture — `audio::capture`

**Responsibility**: Open the preferred CPAL input device, configure it for 16 kHz mono f32, and push raw samples into a `tokio::sync::mpsc` channel.

```rust
// audio/capture.rs

pub struct CaptureHandle {
    pub rx: mpsc::Receiver<Vec<f32>>,
    _stream: cpal::Stream,   // kept alive for the lifetime of recording
}

pub fn start_capture(device: &cpal::Device) -> anyhow::Result<CaptureHandle> {
    // Build StreamConfig: sample_rate=16000, channels=1, SampleFormat::F32
    // In the stream data callback, collect samples into a Vec<f32> and send
    // them through an mpsc channel (bounded, capacity ~4 frames).
    // Resample from the device's native rate if needed (most mics support 16kHz directly).
}
```

**Key decisions**:
- Use `cpal::default_input_device()` for Stage 1; the device-selection command is Stage 6.
- If the device does not natively support 16 kHz, use linear interpolation resampling in the callback. Keep it simple — quality is not critical here; VAD accuracy is.
- The `_stream` field prevents the CPAL stream from being dropped prematurely.

### 2. VAD frame classifier — `audio::vad`

**Responsibility**: Wrap `webrtc-vad`, classify 10 ms frames as speech or silence, and accumulate voiced frames into a growing buffer.

```rust
// audio/vad.rs

use webrtc_vad::{Vad, VadMode, SampleRate};

pub struct VadClassifier {
    vad: Vad,
    voiced_buf: Vec<f32>,       // growing buffer of voiced frames
    silence_run: u32,           // consecutive silent frames
}

impl VadClassifier {
    pub fn new() -> Self {
        let mut vad = Vad::new();
        vad.set_mode(VadMode::Quality);
        Self { vad, voiced_buf: Vec::new(), silence_run: 0 }
    }

    /// Feed one 10 ms frame (160 samples at 16 kHz).
    /// Returns Some(voiced_chunk) when a flush condition is met, None otherwise.
    pub fn push_frame(&mut self, frame: &[f32]) -> Option<Vec<f32>> { ... }
}
```

**Frame sizing**: `webrtc-vad` requires exactly 10 ms frames at 16 kHz = **160 samples**. The CPAL callback may deliver different buffer sizes; the chunker must buffer partial frames and feed them in 160-sample slices.

**Flush conditions** (handled by chunker in §3, triggered from here via return value):
- Silence detected for ≥ 500 ms (50 consecutive silent frames) after voiced content.
- Voiced buffer reaches 30 s (480 000 samples) regardless of VAD state.

**Silence padding**: Retain the last 200 ms of silence frames at chunk boundaries to avoid clipping the trailing edge of speech.

### 4. speech-swift client extension — `client::speech_swift`

**Extend the existing client from Stage 0** to add the `POST /registry/sessions` call:

```rust
// client/speech_swift.rs

#[derive(Debug, serde::Deserialize)]
pub struct SegmentResponse {
    pub speaker_id:    i64,
    pub speaker_label: String,
    pub start_ms:      i64,
    pub end_ms:        i64,
    pub transcript:    String,
}

#[derive(Debug, serde::Deserialize)]
pub struct SessionResponse {
    pub segments: Vec<SegmentResponse>,
}

pub async fn transcribe_chunk(
    base_url: &str,
    wav_bytes: Vec<u8>,
) -> anyhow::Result<SessionResponse> {
    let client = reqwest::Client::new();
    let part = reqwest::multipart::Part::bytes(wav_bytes)
        .file_name("chunk.wav")
        .mime_str("audio/wav")?;
    let form = reqwest::multipart::Form::new().part("audio", part);

    let resp = client
        .post(format!("{}/registry/sessions", base_url))
        .multipart(form)
        .send()
        .await?
        .error_for_status()?
        .json::<SessionResponse>()
        .await?;

    Ok(resp)
}
```

> Verify the exact multipart field name and content-type against the speech-swift API before finalising. Adjust if it expects `application/octet-stream` or a JSON body with base64 audio.

### 5. Embedding model — `embed`

**Responsibility**: Lazy-load `all-MiniLM-L6-v2` once at startup (or on first use), generate 384-dim embeddings for segment text.

```rust
// embed/mod.rs

use fastembed::{TextEmbedding, InitOptions, EmbeddingModel};
use std::sync::OnceLock;

static MODEL: OnceLock<TextEmbedding> = OnceLock::new();

pub fn get_model() -> anyhow::Result<&'static TextEmbedding> {
    MODEL.get_or_try_init(|| {
        TextEmbedding::try_new(InitOptions {
            model_name: EmbeddingModel::AllMiniLML6V2,
            ..Default::default()
        })
    })
}

pub fn embed(text: &str) -> anyhow::Result<Vec<f32>> {
    let model = get_model()?;
    let mut results = model.embed(vec![text], None)?;
    Ok(results.remove(0))
}
```

**Cold-start handling**: The model takes ~1–2 s to load on first call. Segments that arrive before the model is ready are queued and embedded once it is available. Implement this with a `tokio::sync::mpsc` queue in the pipeline loop (see §7).

**Model cache path**: fastembed caches the ONNX model file to `~/.cache/fastembed` by default. No app-specific config needed.

```rust
pub fn insert_segment_embedding(conn: &DbConn, segment_id: i64, embedding: &[f32]) -> anyhow::Result<()> {
    // sqlite-vec expects the embedding serialized as a blob
    let blob = embedding.iter()
        .flat_map(|f| f.to_le_bytes())
        .collect::<Vec<u8>>();
    conn.execute(
        "INSERT INTO segment_embeddings (segment_id, embedding) VALUES (?1, ?2)",
        rusqlite::params![segment_id, blob],
    )?;
    Ok(())
}
```

```rust
pub fn upsert_speaker(
    conn: &DbConn,
    speech_swift_id: i64,
    now_ms: i64,
) -> anyhow::Result<(Speaker, bool)> {
    let existing = conn.query_row(
        "SELECT id, speech_swift_id, display_name, notes, first_seen_at, last_seen_at
         FROM speakers WHERE speech_swift_id = ?1",
        [speech_swift_id],
        row_to_speaker,
    ).optional()?;

        conn.execute(
            "UPDATE speakers SET last_seen_at = ?1 WHERE speech_swift_id = ?2",
            [now_ms, speech_swift_id],
        )?;
        Ok((speaker, false))

        conn.execute(
            "INSERT INTO speakers (speech_swift_id, first_seen_at, last_seen_at)
             VALUES (?1, ?2, ?3)",
            [speech_swift_id, now_ms, now_ms],
        )?;
        let speaker = conn.query_row(
            "SELECT id, speech_swift_id, display_name, notes, first_seen_at, last_seen_at
             FROM speakers WHERE speech_swift_id = ?1",
            [speech_swift_id],
            row_to_speaker,
        )?;
        Ok((speaker, true))
    }
}
```

### 7. Pipeline task — the core loop

This is the central piece. When `start_session` is called, a `tokio::task` is spawned that:

1. Opens the CPAL capture stream.
2. Feeds samples through the `Chunker`.
3. For each flushed chunk: saves WAV to disk, calls speech-swift, persists all results, generates embeddings, emits events.

```rust
// commands/mod.rs (pipeline task, simplified)

async fn run_pipeline(
    session_id: i64,
    app_handle: tauri::AppHandle,
    stop_rx: oneshot::Receiver<()>,
) {
    let preferred = state.preferred_device.lock().unwrap().clone();
    let CaptureHandle { mut rx, _stream } =
        audio::capture::start_capture(preferred.as_deref()).unwrap();
    let mut chunker = Chunker::new();

    // Warm up the embed model in the background so it's ready when first chunk arrives
    tokio::task::spawn_blocking(|| { let _ = embed::get_model(); });

    // Pending segments waiting for embed model (cold-start queue)
    let mut embed_queue: Vec<(i64, String)> = Vec::new();

    tokio::select! {
        _ = stop_rx => { /* session stopped */ }
        _ = async {
            while let Some(samples) = rx.recv().await {
                if let Some((wav_bytes, start_ms, end_ms)) = chunker.push_samples(&samples) {
                    handle_chunk(
                        session_id, wav_bytes, start_ms, end_ms,
                        &app_handle, &mut embed_queue
                    ).await;
                }
            }
        } => {}
    }

    // Flush any remaining voiced samples on stop
    if let Some((wav_bytes, start_ms, end_ms)) = chunker.flush() {
        handle_chunk(session_id, wav_bytes, start_ms, end_ms, &app_handle, &mut embed_queue).await;
    }

    // Drain embed queue for any segments that arrived during model cold-start
    drain_embed_queue(embed_queue, &app_handle).await;
}
```

**`handle_chunk`** (the per-chunk handler):

```rust
async fn handle_chunk(
    session_id: i64,
    wav_bytes:  Vec<u8>,
    start_ms:   u64,
    end_ms:     u64,
    app:        &tauri::AppHandle,
    embed_queue: &mut Vec<(i64, String)>,
) {
    // 1. Save WAV to disk
    let audio_path = save_wav_chunk(app, session_id, start_ms, &wav_bytes);

    // 2. POST to speech-swift
    let state = app.state::<AppState>();
    let base_url = &state.speech_swift_url;
    let response = match client::speech_swift::transcribe_chunk(base_url, wav_bytes).await {
        Ok(r) => r,
        Err(e) => { eprintln!("speech-swift error: {e}"); return; }
    };

    let now_ms = unix_ms();
    let db = state.db.lock().unwrap();

    for seg in &response.segments {
        // 3. Upsert speaker
        let (speaker, is_new) = db::speakers::upsert_speaker(&db, seg.speaker_id, now_ms).unwrap();

        // 4. Insert segment
        let segment_id = db::segments::insert_segment(&db, &NewSegment {
            session_id,
            speaker_id: seg.speaker_id,
            start_ms:   seg.start_ms,
            end_ms:     seg.end_ms,
            transcript_text: seg.transcript.clone(),
        }).unwrap();

        // 5. Insert speaker_sample (one per speaker per chunk)
        let _ = db::samples::insert_speaker_sample(
            &db, speaker.id, session_id,
            seg.start_ms, seg.end_ms, &audio_path,
        );

        // 6. Embed (or queue if model not ready)
        let text = seg.transcript.clone();
        match embed::embed(&text) {
            Ok(vec) => {
                let _ = db::segments::insert_segment_embedding(&db, segment_id, &vec);
            }
            Err(_) => {
                embed_queue.push((segment_id, text));
            }
        }

        // 7. Emit segment_added
        events::emit_segment_added(app, FullSegment { segment_id, speaker: speaker.clone(), ..seg.into() });

        // 8. Emit new_speaker if unnamed
        if is_new || speaker.display_name.is_none() {
            events::emit_new_speaker(app, speaker);
        }
    }
}
```

**WAV chunk persistence** (`save_wav_chunk`):

```rust
fn save_wav_chunk(app: &tauri::AppHandle, session_id: i64, start_ms: u64, bytes: &[u8]) -> String {
    let dir = app.path().app_data_dir().unwrap()
        .join("audio_chunks")
        .join(session_id.to_string());
    std::fs::create_dir_all(&dir).unwrap();
    let path = dir.join(format!("{start_ms}.wav"));
    std::fs::write(&path, bytes).unwrap();
    path.to_string_lossy().into_owned()
}
```

```rust
#[derive(serde::Serialize, Clone)]
pub struct SegmentEvent {
```

> **Type note**: `Speaker` (the full DB struct in `db::speakers`) has 6 fields; `SpeakerEvent` (the event payload) has 3. `SpeakerWithStats` (Stage 3, the `get_speakers` command result) has 7. ### 9. Tauri commands — `commands::mod`

```rust
// commands/mod.rs

use tokio::sync::oneshot;
use std::collections::HashMap;
use std::sync::Mutex;

// Stored in AppState: maps session_id → stop sender
pub type PipelineRegistry = Mutex<HashMap<i64, oneshot::Sender<()>>>;

#[tauri::command]
pub async fn start_session(
    app: tauri::AppHandle,
    state: tauri::State<'_, AppState>,
) -> Result<i64, String> {
    let now_ms = unix_ms();
    let session_id = {
        let db = state.db.lock().unwrap();
        db.execute(
            "INSERT INTO sessions (created_at, source) VALUES (?1, 'mic')",
            [now_ms],
        ).map_err(|e| e.to_string())?;
        db.last_insert_rowid()
    };

    let (stop_tx, stop_rx) = oneshot::channel();
    state.pipelines.lock().unwrap().insert(session_id, stop_tx);

    let app_clone = app.clone();
    tauri::async_runtime::spawn(async move {
        run_pipeline(session_id, app_clone, stop_rx).await;
    });

    Ok(session_id)
}

#[tauri::command]
pub async fn stop_session(
    session_id: i64,
    state: tauri::State<'_, AppState>,
) -> Result<(), String> {
    let stop_tx = state.pipelines.lock().unwrap().remove(&session_id);
    if let Some(tx) = stop_tx {
        let _ = tx.send(());
    }

    // Finalize duration_ms
    let now_ms = unix_ms();
    let db = state.db.lock().unwrap();
    db.execute(
        "UPDATE sessions SET duration_ms = ?1 - created_at WHERE id = ?2",
        rusqlite::params![now_ms, session_id],
    ).map_err(|e| e.to_string())?;

    Ok(())
}
```

## Threading Model

```
tokio runtime (Tauri's async runtime)
│
├── main thread: Tauri event loop
│
├── pipeline task (per active session):
│   ├── CPAL callback thread (OS-managed, sends to mpsc)
│   └── async loop: recv samples → chunker → handle_chunk
│       └── handle_chunk awaits: speech-swift HTTP call
│           then synchronously: DB writes + embed
│
└── embed warm-up task: spawn_blocking → loads ONNX model
```

**Key constraint**: `rusqlite::Connection` is `!Send`. Wrap it in `Mutex<Connection>` in `AppState` and lock it only for the duration of each write. Never `.await` while holding the lock — complete all async work (HTTP call) before acquiring the lock for DB writes.

## Completion Criteria

| # | Check |
|---|---|
| 1 | `start_session` returns a valid `session_id`; row appears in `sessions` table |
| 2 | Speaking into the mic for 5 s and stopping produces at least one row in `segments` |
| 3 | Each segment row has a non-empty `transcript_text` and a valid `speaker_id` |
| 4 | Each segment has a corresponding row in `segment_embeddings` with a 384-element blob |
| 5 | Each chunk's WAV file exists on disk at the path stored in `speaker_samples.audio_path` |
| 6 | `speakers` table has one row per unique `speaker_id` seen |
| 7 | `stop_session` sets a non-null `duration_ms` on the session row |
| 8 | `segment_added` event fires in the Tauri developer console for each segment |
| 9 | `new_speaker` event fires the first time each speaker is seen |
| 10 | Two back-to-back sessions do not share pipeline state (stop one, start another — clean slate) |
| 11 | Cold-start: segments arriving before fastembed loads are embedded after model is ready (check `segment_embeddings` row count equals `segments` row count after session ends) |
| 12 | `cargo build` zero errors, zero warnings |
